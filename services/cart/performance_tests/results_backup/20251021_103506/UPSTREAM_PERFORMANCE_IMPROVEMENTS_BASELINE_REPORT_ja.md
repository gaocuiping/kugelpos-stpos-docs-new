### 定常状態統計（Steady-State Metrics）

定常状態におけるレスポンスタイムの変動性を測定し、システムの一貫性を評価します。

| ユーザー数 | 平均RT (ms) | 標準偏差 (ms) | CV (%) | 評価 |

| **30** | 76.15 | 4.63 | 6.07 | ✅ 優秀 |
| **40** | 75.68 | 4.06 | 5.36 | ✅ 非常に優秀 |
| **50** | 68.72 | 3.10 | 4.51 | ✅ 非常に優秀 |

**CV (変動係数) = (標準偏差 / 平均) × 100**

### 一貫性評価

CV値が低いほど一貫したパフォーマンスを示します：
- **< 10%**: 優秀な一貫性
- **10-20%**: 良好な一貫性
- **20-30%**: 許容範囲
- **> 30%**: 改善が必要

**分析:**

✅ **非常に優れた一貫性**
- すべての負荷レベルでCV値が10%未満
- 50ユーザーで最も一貫性が高い（CV 4.51%）
- 標準偏差は3-5ms範囲 - 非常に安定

✅ **負荷増加に伴う一貫性向上**
- 20ユーザー: CV 7.07%
- 30ユーザー: CV 6.07%
- 40ユーザー: CV 5.36%
- 50ユーザー: CV 4.51%
- 高負荷ほど予測可能なパフォーマンス

✅ **Fire-and-forget logging の効果**
- DB書き込み遅延がレスポンスタイムに影響していない
- 低い変動性は最適化の成功を示す

**結論:**
システムは全負荷レベルで**優れた一貫性**を示しています。特に高負荷（40-50ユーザー）でのCV値4-5%台は、商用システムとして非常に優秀なレベルです。

## 技術的分析

### 適用された変更内容

#### 1. Fire-and-forget logging (d94748e)

**ファイル:** `services/commons/src/kugel_common/middleware/log_requests.py`

**実装内容:**
```python
# DB書き込みをバックグラウンドタスク化
asyncio.create_task(_output_request_log_to_db_async(...))
```

**効果:**
- API レスポンスが DB 書き込み完了を待たない
- P99 レイテンシが 1秒未満に改善
- DB 書き込みエラーがリクエストに影響しない

#### 2. HTTP connection pooling (506a2c3)

**ファイル:** `services/commons/src/kugel_common/security.py`

**実装内容:**
```python
client = await get_pooled_client("terminal")
response_data = await client.get(
    f"/terminals/{terminal_id}",
    headers={"X-API-KEY": api_key}
)
```

**効果:**
- TCP 3-way handshake のオーバーヘッド削減
- SSL/TLS ハンドシェイク削減
- コネクション再利用による効率化

#### 3. Commons バージョンアップ

- **バージョン:** v0.1.33
- **変更:** HTTP connection pooling、fire-and-forget logging を含む
